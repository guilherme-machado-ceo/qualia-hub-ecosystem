## Definição

Engine de matching semântico de propósito geral entre sistemas formais
heterogêneos. Opera por iteração inteligente e matching contextual,
preservando semântica através de transformações entre domínios distintos.

Distingue-se de transpilers tradicionais por operar em nível semântico
e ontológico, não apenas sintático.

---

## GuruMatrix 5D — Os Cinco Eixos Paramétricos

Cada token processado pela IPII é indexado em cinco dimensões:

| Eixo | Símbolo | Descrição | Exemplos |
|------|---------|-----------|---------|
| Categoria Ontológica | **i** | Natureza do elemento | Substância, Ação, Relação, Qualidade |
| Campo do Conhecimento | **j** | Domínio semântico | Direito, Tecnologia, Arte, Ciência |
| Nível Hermenêutico | **k** | Profundidade interpretativa | Literal → Sistemático → Teleológico → Ontológico |
| Tempo de Execução | **t** | Regime temporal | Compilação, Vigência, Deadline, Condicional |
| Paradigma-Alvo | **l** | Âmbito de destino | Python, Federal, Estadual, PQC, DIH |

## Fluxo Operacional

```
INPUT: token fonte + contexto
       ↓
1. TOKENIZAÇÃO SEMÂNTICA
   Identificar tokens no sistema fonte
   Construir AST semântico-ontológica
       ↓
2. INDEXAÇÃO LEXMATRIX
   Mapear cada token nos 5 eixos (i, j, k, t, l)
   Resultado: vetor dimensional do token
       ↓
3. ITERAÇÃO INTELIGENTE
   Consultar banco pan-domínio
   Encontrar equivalências no sistema alvo
   Iterar sobre múltiplas opções
       ↓
4. MATCHING CONTEXTUAL
   Selecionar melhor mapeamento
   Critérios: precisão semântica, conformidade, legibilidade
       ↓
OUTPUT: token alvo com preservação de significado
```

## Protocolo de Instanciação em Novo Domínio

1. **Identificar sistemas formais** — quais dois sistemas precisam
   se comunicar?
2. **Mapear nos 5 eixos** — categorizar os tokens do domínio
3. **Construir banco pan-domínio** — catalogar equivalências
4. **Implementar matching contextual** — heurísticas de seleção
5. **Documentar** em `instances/[nome]/`

## Vantagens sobre sistemas tradicionais

- **Precisão semântica**: preserva intenção, não apenas sintaxe
- **Interoperabilidade nativa**: sem wrappers ou FFIs ad-hoc
- **Flexibilidade**: novo domínio = atualizar banco + treinar modelo
- **Multimodal**: texto, código, normas jurídicas, protocolos quânticos
- **Base para IAG**: compreensão semântica cross-domain
